# Header Positioning Design

## Goal

Reposition the CV and ATS headers so the profile is still led by `QA Lead`, while clearly signaling `Senior Full Stack Engineer` and `AI-Native Developer` as the next two identity markers.

## Context

The current headers and summaries still lean too heavily toward QA automation / DevOps phrasing. That underrepresents the current hands-on full stack and AI-native engineering work, especially around `Lexense.ai`.

The user wants:

- `QA Lead` to remain first
- `Senior Full Stack Engineer` second
- `AI-Native Developer` third

This change is about top-level positioning only. Historical role titles should stay accurate, especially the `Quality Assurance Team Lead` role in `Humanit`.

## Preferred Positioning

Use the same ordering across all public-facing variants:

- `QA Lead`
- `Senior Full Stack Engineer`
- `AI-Native Developer`

## Messaging Principles

### 1. Keep the profile truthful

`QA Lead` stays first because it reflects the largest block of experience and the current formal role at `Humanit`.

### 2. Change first impression without rewriting history

The headers and summary should make the reader immediately understand that the candidate is not only a QA specialist, but also a senior full stack engineer who actively builds AI-native systems.

### 3. Do not rename past job titles

The experience section remains factually accurate:

- `Humanit` stays `Quality Assurance Team Lead`
- legacy QA titles remain unchanged

Only the profile header and summary framing are adjusted.

### 4. Remove narrow `QA Automation Engineer` framing from the header

That phrase is too narrow for the intended positioning and should be replaced with `Senior Full Stack Engineer`.

### 5. Align summary with the new header

The summary should start from `QA Lead` but explicitly include:

- senior full stack engineering
- AI-native development
- hands-on Node.js / Python / Django work
- platform, AI, and delivery ownership

## Variant-Specific Design

### Full CV

Header:

- `QA Lead · Senior Full Stack Engineer · AI-Native Developer`

No long summary block exists in the full CV header, so only the title line changes.

### Resume

Header:

- `QA Lead · Senior Full Stack Engineer · AI-Native Developer`

Summary:

- Start from `QA Lead`
- Add `Senior Full Stack Engineer` and `AI-Native Developer`
- Keep the rest concise and high-signal

### ATS

Header:

- `QA Lead | Senior Full Stack Engineer | AI-Native Developer`

Summary:

- Replace the current `QA Lead and DevOps Engineer` framing
- Use engineering-first wording after `QA Lead`
- Avoid `QA Automation Engineer`

## Out of Scope

- Rewriting role titles in work experience
- Repositioning `Humanit` away from QA leadership
- Large changes to skills taxonomy
- Structural layout changes outside what is needed for text fit

## Acceptance Criteria

- `QA Lead` remains the first identity marker everywhere.
- `Senior Full Stack Engineer` and `AI-Native Developer` are visible in the main profile headers.
- `QA Automation Engineer` is removed from the main ATS header.
- Resume and ATS summaries match the new positioning.
- Experience titles remain historically accurate.
